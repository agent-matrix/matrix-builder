# Public Website Pages

Matrix Builder should be promoted through `ruslanmv.com` and connected to MatrixHub.

## ruslanmv.com

Recommended routes:

```text
/matrix-builder
/definitions
/definitions/current
/docs/ai-coder-contract
/examples
/matrixhub
```

## matrixhub.io

Recommended routes:

```text
/
/bundles
/templates
/blueprints
/publish
/docs/matrix-bundles
```

## Route responsibilities

| Route | Responsibility |
|---|---|
| `ruslanmv.com/matrix-builder` | Product doorway and builder UI |
| `ruslanmv.com/definitions` | Ruslan Magana Definitions public standard |
| `ruslanmv.com/docs/ai-coder-contract` | Explain controlled generation |
| `matrixhub.io` | Registry and discovery |
| `matrixhub.io/publish` | Publish validated Matrix Bundles |

## Cross-linking

Every Matrix Builder bundle page should link to:

- agent-generator engine docs
- matrix-definitions pack version
- Ruslan Magana Definitions page
- MatrixHub publish/dry-run status
- GitPilot prompt path
